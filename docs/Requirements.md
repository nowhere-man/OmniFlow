# OmniFlow Requirements

本文档定义产品需求。验收标准见 `docs/Goal.md`，技术实现见 `docs/Architecture.md`，历史讨论记录见 `docs/DiscussionLog.md`。

---

## 1. 产品定位

- 一款面向个人的多平台记账 App，长期目标是替换「青子记账」。
- 过渡期需支持与青子记账数据互通（双向导入导出），避免迁移期被单一 App 绑定。
- 首版只做单用户，必须支持多账本。
- 需求讨论覆盖完整产品目标，实现可以分阶段推进。

---

## 2. 平台范围

| 平台 | 支持 | 备注 |
|------|------|------|
| iOS | ✅ | 首版以 iPhone 为主，iPad 不单独设计 |
| macOS | ✅ | 桌面端，需支持侧边栏、菜单栏、快捷键、窗口布局 |
| Android | ✅ | 首版只支持手机，平板/折叠屏不单独设计 |
| Web / PWA | ❌ | 不支持 |
| Windows | ❌ | 暂不纳入，除非后续重新确认 |

- Apple 端（iOS + macOS）使用一个工程，共享 SwiftUI 基础组件、设计 token 和图表数据模型，但导航、窗口布局、快捷键等按平台分别设计。
- Android 与 Apple 允许维护两套 UI 代码，以换取更好的平台体验。
- 目标是精美、成熟、符合各平台主流设计习惯，而非 100% 使用系统原生控件。

---

## 3. 基础记账

- 支持手动新增交易，字段包含：金额、分类、账户、备注、商户、标签、日期。
- 交易类型仅支持**收入**和**支出**两种，不存在第三种类型。
- 每条交易必须归属于一个账本。
- 手动新增交易必须选择分类，分类从当前账本的分类列表中选择，不允许自由文本填写。
- 手动新增交易不允许保存为未分类。
- 支持编辑已有交易的任意字段。
- 支持删除交易（软删除）。
- 支持将一笔交易标记为「不计入收支」，标记后该笔交易在统计图表中不参与计算。
  - 适用场景：转账、退款、还款等不应污染真实收支的明细。
- 退款处理策略：保持极简，未跨月退款可删原单，跨月退款手工记一笔"收入"并备注退款。不引入复杂系统平账。
- 预算功能首版不做。

---

## 4. 多账本

- 支持创建、编辑、删除多个账本。
- 首版只支持单用户多账本，不支持家庭共享账本、多人协作、成员权限或共享邀请。
- 首次启动时自动创建一个「默认账本」。
- 新建账本时自动复制系统预置的收入/支出一级分类模板。
- 新建账本时不复制默认规则，不自动创建账户。

---

## 5. 账户管理

- 账户是全局共享资源，不从属于任何账本，所有账本共用同一套账户池。
- 账户类型支持：现金、储蓄卡、信用卡、电子钱包、其他。
- 支持创建、编辑、删除账户。
- 首次启动时自动创建一个全局「现金」账户。
- 账户可维护一个用户手动填写的当前余额。
- 账户余额不要求从交易流水自动推导，不要求与交易流水精确平账。
- 余额精度说明：信用卡还款、跨行转账等可不记账，忽略账户余额偏差，不引入"流水关联"机制。
- 还款提醒不绑定账户，账户模型不包含还款提醒字段。

---

## 6. 分类与标签

- 分类按账本独立，不全局共享。
- 分类为两级树结构：一级分类（父）和二级分类（子）。
- 系统内置以下一级分类模板，新建账本时复制，用户可在账本内修改：
  - **支出类**：餐饮美食、交通出行、日用百货、居住服务、休闲娱乐、医疗健康、教育培训、充值缴费、其他支出。
  - **收入类**：工资、投资理财、兼职副业、奖金红包、其他收入。
- 每个分类有固定的类型属性（支出或收入），不可混用。
- 标签按账本独立，不全局共享。
- 标签为扁平多标签系统，一条交易可以打多个标签，也可以不打标签。

---

## 7. 外部账单导入

### 7.1 支持的导入来源

| 来源 | 文件格式 | 编码 |
|------|---------|------|
| 支付宝 | CSV | GBK |
| 微信 | XLSX | UTF-8 |
| 京东 | CSV | UTF-8 BOM |
| 美团 | CSV | UTF-8 BOM |
| 建设银行 | XLS (BIFF8) | — |

- 不支持 PDF 解析。
- 不支持抖音（已移除）。

### 7.2 通用导入规则

- 导入前必须选择目标账本，导入会话中的明细不得被规则改派到其他账本。
- 所有导入明细默认归属全局「现金」账户，首版不支持按账单来源设置默认账户。
- 导入阶段不自动根据支付平台自带分类映射 App 分类。
- 「不计收支」类交易照常导入，自动设置 `is_excluded = true`，不跳过、不丢弃。
- 确认入账后不保留导入批次记录、原始解析明细快照或上传的原始文件。
- 首版不支持撤销上一次导入。
- 最终交易必须保留 `external_source` 和 `external_id`，用于后续重复导入去重。

### 7.3 建设银行特殊规则

- 收支判断：金额正数 = 收入，金额负数 = 支出；摘要字段辅佐验证。
- 时间精度仅到天（格式 `20260601`），无时分秒，跨平台模糊去重时需特殊处理。

---

## 8. 规则引擎

- 规则按账本独立，不全局共享。
- 规则只在导入预览阶段执行，不对已入账历史交易批量重跑。
- 规则不支持把明细改派到其他账本。
- 支持的匹配条件：解析出的文本字段包含关键字，字段来源包括商户名、商品说明、备注、交易对方、交易类型、来源平台。
- 支持的执行动作：设置一级分类、设置二级分类、改账户、加标签、设为不计入收支、排除不入账。
- 规则按用户定义的优先级顺序执行，命中后短路（不再匹配后续规则）。
- 支持规则的 CRUD。
- 用户可在导入预览页覆盖规则结果，最终以用户确认的预览结果入账。

---

## 9. 自动分类记忆

- 导入自动分类优先级：**账本内规则 > 同账本历史分类记忆 > 用户手动选择**。
- 历史分类记忆只作为规则未命中时的兜底，不替代用户自定义规则。
- 规则和历史分类记忆都未命中时，预览项分类为空，用户必须手动选择后才能入账。
- 用户在导入预览页手动选择分类后，该选择进入同账本历史分类记忆，后续相同或相似明细可自动填分类。

---

## 10. 去重引擎

- **绝对去重**：相同 `external_id` + `external_source` 的交易直接标记为重复，不入账，用户无法强行入账。
- **模糊去重**：时间窗口（±2 小时）+ 金额相等 + 商户名相似度超过阈值 → 标记为「疑似重复」。
- 建设银行来源因时间仅精确到天，模糊去重窗口放宽为同日。
- 疑似重复项默认不入账，用户手动确认保留后才允许入账。

---

## 11. 导入预览流程（一站式预览页）

- 用户上传文件后，系统自动完成：解析 → 规则匹配 → 去重检测。
- 预览页展示完整的待入账明细列表。
- 重复项（绝对重复和疑似重复）有醒目标识。
- 用户可在预览页逐条编辑（修改分类、账户、标签、备注、是否排除）。
- 用户可在预览页批量操作（全选/反选、批量改分类、批量排除）。
- 存在分类为空且未排除的预览项时，不允许一键入账。
- 用户确认后一键入账，所有通过确认的明细写入数据库。

---

## 12. 数据分析与图表

- 支持精美、丰富的原生图表，用于分析记账数据。
- 图表类型：收支趋势（折线图）、分类占比（饼图/环形图）、账户资产分析、同环比分析、标签分析、收支排行榜。
- 所有图表支持按天、周、月、年、自定义时间范围过滤。
- 统计结果自动排除 `is_excluded = true` 的交易。
- 分析口径在 KMP 共享层统一维护，不同平台只负责按各自设计语言渲染。
- iOS/macOS 使用 Swift Charts 或 Apple 原生绘制能力，不使用 WebView 图表。
- Android 使用 Compose 图表库或 Canvas，不使用 WebView 图表。
- 图表支持触点/鼠标选择、图例切换、时间范围切换、分类下钻和筛选联动。

---

## 13. 高级搜索与聚合

- 支持按以下维度任意组合筛选：一级分类、二级分类、标签、账户、商户关键字、金额范围、时间范围。
- 搜索结果页自动计算并展示筛选范围内的总收入和总支出。

---

## 14. 提醒与周期事项

- 提醒是独立功能，不绑定账本、账户或交易。
- 提醒不自动生成交易，不参与记账统计，不影响账户余额。
- 提醒参与多端加密同步。
- 提醒使用本地系统通知，不做服务器推送。
- 提醒分为两个子类型：
  - **还款提醒**：金额可选（仅用于展示），支持固定还款日期或账单日后 N 天计算。
  - **订阅提醒**：金额可选（用于展示和未来固定支出概览），支持每天/每周/每月/每年重复。
- 首版不支持复杂 RRULE、法定节假日顺延或工作日规则。
- 首版提醒不生成待确认记账卡片，不与导入交易自动合并。

---

## 15. 青子记账互通

- 支持导入青子记账导出的 JSON 文件，导入后进入标准导入流水线（规则 → 去重 → 预览 → 入账）。
- 支持将本项目数据导出为青子记账兼容的 JSON 格式。
- 互通等级目标：**长期双写**——过渡期两个 App 同时使用，通过导入导出实现双向数据合并与增量同步。
- 字段映射规则：
  - `entry.value` → 金额
  - `entry.createDate`（Unix 时间戳）→ 交易时间
  - `entry.content` → 备注
  - `category.type`（0=收入，1=支出）→ 交易类型
  - `entry.categoryID` → 关联分类名称
  - `entry.accountID` → 关联账户名称
  - `entry.markIDs`（逗号分隔标签 ID）→ 标签文本数组
  - `entry.excludeFromBudget = true` → `is_excluded = true`
- 无法映射的字段（如位置信息、退款周期）在导入时直接丢弃，记录警告，不阻断流程。
- 支持基于时间范围或重复检测的增量操作。

---

## 16. 数据同步

- 同步采用本地优先模型：无网络时所有功能正常工作，有网络时自动/手动同步。
- 远端第一阶段只作为加密同步存储，不承载明文业务数据、查询、统计、导入解析或规则执行。
- 首选 WebDAV 作为同步后端；内部抽象为「同步适配器」，未来可扩展 iCloud、坚果云、S3、自建服务等，不改变业务层接口。
- 上传到远端的数据默认加密，远端服务无法读取账本、交易、分类、标签和统计内容。
- 本地设备负责解密和使用数据，加密密钥不由远端持有。

---

## 17. 首版不做的功能

| 功能 | 说明 |
|------|------|
| 家庭共享账本 | 首版只做单用户 |
| 多人协作 / 成员权限 | 同上 |
| 预算管理 | 首版不做 |
| 撤销导入 | 首版不支持 |
| 按账单来源设置默认账户 | 全部默认「现金」 |
| 周期事项自动生成交易 | 提醒模块只提醒，不记账 |
| 复杂 RRULE / 节假日顺延 | 提醒只支持简单重复 |
| Web / PWA / Windows | 不在平台范围内 |
| iPad / 平板 / 折叠屏专属设计 | 首版不单独设计 |
| PDF 账单解析 | 不支持 |
| 抖音账单导入 | 已移除 |
| 服务器推送通知 | 只用本地通知 |
